# 🚀 方案 B：Sealos 云部署指南 — 扬州培训平台

## 你将获得

`https://xxx.sealos.run` 免费域名 + 自动 HTTPS + 数据持久化

全程通过 **Sealos DevBox** 云端完成，**无需本地安装 Docker**。

---

## 准备工作

你已经有一个准备好的部署包：**`yz-training-deploy.zip`**（约 154KB）

里面包含：所有源代码 + Dockerfile + 部署脚本 `deploy-in-devbox.sh`

---

## 第一步：注册 Sealos（2 分钟）

1. 浏览器打开 **https://cloud.sealos.run**
2. 右上角点击「注册」，支持 **微信扫码** 或 **手机号注册**
3. 注册后自动赠送 **5 元免费额度**（0.5核/512MB 配置可用一周以上）

---

## 第二步：创建 DevBox（3 分钟）

1. 登录后，左侧菜单点击 **「DevBox」**
2. 点击 **「新建 DevBox」**
3. 填写配置：

| 参数 | 值 |
|------|-----|
| 项目名称 | `yz-training` |
| 模板 | **Ubuntu 22.04**（选有 Docker 的） |
| CPU | 2 核 |
| 内存 | 4 GB |

4. 点击「创建」，等待 30-60 秒启动
5. 点击 **「打开」**，进入在线 VS Code 编辑器

---

## 第三步：上传项目文件（1 分钟）

在 DevBox 的 VS Code 中：

1. 左侧文件浏览器，右键空白处 → **「上传」**
2. 选择你电脑上的 **`yz-training-deploy.zip`** 文件
3. 等待上传完成（文件显示在左侧列表即可）

---

## 第四步：运行部署脚本（5 分钟）

在 DevBox 底部打开 **终端（Terminal）**，执行：

```bash
# 解压并运行部署脚本
unzip yz-training-deploy.zip
chmod +x deploy-in-devbox.sh
bash deploy-in-devbox.sh
```

脚本会自动完成：
1. ✅ 检查/安装 Docker
2. ✅ 解压项目代码
3. ✅ 安装 Node.js 依赖
4. ✅ 构建 Docker 镜像
5. ✅ 推送到 Sealos 镜像仓库

运行过程中脚本会询问你的 **Sealos 用户名（手机号）**，输入即可。

---

## 第五步：部署到应用管理（2 分钟）

脚本执行完成后，回到 Sealos 网页控制台：

1. 左侧菜单 → **「应用管理」**
2. 点击 **「新建应用」**

### 基础配置

| 参数 | 值 |
|------|-----|
| 应用名称 | `yz-training-platform` |
| 镜像地址 | `sealos.hub/你的手机号/yz-training:latest` |
| CPU | 0.5 核 |
| 内存 | 512 MB |

### 网络配置

| 参数 | 值 |
|------|-----|
| 容器端口 | `3001` |
| 外网访问 | ✅ **开启**（自动分配域名） |

### 环境变量

| 变量名 | 值 |
|--------|-----|
| `DATA_DIR` | `/data` |
| `UPLOAD_DIR` | `/data/uploads` |

### 持久化存储

| 参数 | 值 |
|------|-----|
| 挂载路径 | `/data` |
| 存储大小 | 1 GB |

3. 点击 **「部署应用」**，等待 1-2 分钟

---

## 🔧 故障排查

| 问题 | 解决 |
|------|------|
| DevBox 中 `docker: command not found` | 运行 `sudo apt update && sudo apt install -y docker.io` |
| `docker push` 报 403 | 先运行 `docker login sealos.hub`，用户名用手机号 |
| 部署后访问 502 | 检查环境变量 `DATA_DIR=/data` 是否正确 |
| 数据重启后丢失 | 检查持久卷挂载 `/data` 是否已配置 |
| DevBox 打不开 | 刷新页面或重新创建 |
| 镜像地址找不到 | 在脚本输出中确认完整地址，或 `docker images` 查看 |
